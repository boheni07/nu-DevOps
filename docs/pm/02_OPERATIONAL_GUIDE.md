# 운영 가이드 (Operational Guide)

시스템 배포 후 안정적인 서비스 유지를 위한 운영 및 장애 대응 매뉴얼입니다.

## 1. 모니터링 시스템
- **성능 모니터링**: 서버 CPU, 메모리 사용량 및 API 응답 시간 실시간 감시.
- **에러 로그**: 가급적 실시간 에러 트래킹 도구(Sentry 등)를 연동하여 장애 상황 즉시 파악.

## 2. 장애 대응 절차
1.  **발견**: 모니터링 알람 또는 사용자 제보를 통해 장애 인지.
2.  **전파**: 유관 부서 및 책임자에게 장애 상황 공유.
3.  **원인 분석**: 로그 분석을 통한 문제 지점 파악.
4.  **조치**: 긴급 패치 배포 또는 인프라 확장 등의 조치 수행.
5.  **사후 보고**: 장애 원인 및 재발 방지 대책을 포함한 장애 보고서 작성.

## 3. 백업 및 복구
- **DB 백업**: 매일 새벽 자동 백업을 수행하며 최소 30일간의 데이터를 보관합니다.
- **복구 테스트**: 분기별 1회 이상 백업 데이터를 활용한 복구 시뮬레이션을 수행하여 백업의 정합성을 확인합니다.
