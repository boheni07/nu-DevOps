# 표준 WBS 템플릿 (Project WBS Standard)

프로젝트 규모와 유형에 따른 표준 업무 분업 구조(WBS) 가이드라인입니다.

## 1. 표준 WBS 구조 (기본형)

### Phase 1: Planning & Analysis (기획 및 분석)
- 요구사항 정의 (고객 인터뷰, 요구사항 분석서 작성)
- 시스템 기능 정의 (기능 상세 분석, 유즈케이스 작성)
- UI/UX 프로토타이핑 (와이어프레임 설계, 디자인 컨펌)

### Phase 2: Design (상세 설계)
- 데이터베이스 설계 (ERD, 테이블 정의서)
- 아키텍처 및 인터페이스 설계 (API 명세서 작성)
- 상세 화면 설계 (스토리보드 확정)

### Phase 3: Development (구현)
- 개발 환경 설정 및 공통 모듈 구현
- 기능별 단위 개발 (Front/Back)
- 단위 테스트 수행

### Phase 4: Verification (검증)
- 통합 및 E2E 테스트
- 보안 취약점 점검 및 성능 최적화
- 사용자 수락 테스트 (UAT)

### Phase 5: Deployment & Stabilization (배포 및 안정화)
- 운영 환경 배포 및 초기 데이터 마이그레이션
- 사용자 매뉴얼 교육 및 시스템 안정화 모니터링

## 2. WBS 핵심 관리 지표
- **진척률 관리**: 하위 태스크의 진척률이 상위 태스크에 실시간 반영되어야 함.
- **일정 준수**: Critical Path 상의 태스크를 식별하고 지연 시 즉시 리소스 재배치 검토.
- **보고 체계화**: 일간/주간/월간 단위의 성과 리포트 제출 및 PM 승인 프로세스를 통해 데이터 기반의 진행 상황 모니터링 수행.
