# 품질 보증 계획 (Test Plan)

시스템의 안정성을 보장하기 위한 단계별 테스트 전략입니다.

## 1. 테스트 계층 구조

### Unit Test (단위 테스트)
- **도구**: Jest / React Testing Library
- **대상**: 개별 함수, 공통 컴포넌트, 유틸리티 로직의 정확성 검증.

### Integration Test (통합 테스트)
- **대상**: 여러 컴포넌트 간의 상호작용 및 API 연동 상태 확인.

### E2E Test (End-to-End 테스트)
- **도구**: Cypress 또는 Playwright
- **대상**: 실제 사용자 시나리오 (로그인 -> 프로젝트 생성 -> 업무 등록)를 처음부터 끝까지 자동화 테스트.
- **보고 시스템 시나리오**: 리포트 생성 -> 업무 로그 자동 매핑 확인 -> 종합 의견 작성 -> 보고하기(Submit) -> 작성 후 수정 제한 확인 -> PM 승인 처리 프로세스 검증.

## 2. 테스트 관리 프로토콜
- **Regression Test**: 신규 기능 추가 시 기존 기능에 영향이 없는지 전수 체크.
- **Code Coverage**: 핵심 비즈니스 로직에 대해 최소 80% 이상의 테스트 커버리지를 유지할 것을 권장합니다.
